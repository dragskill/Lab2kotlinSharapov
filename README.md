Протокол лабораторної роботи №2
Тема

Списки та управління станом у Jetpack Compose.

Варіант

Варіант 3 — «Витрати за день».

Мета роботи

Метою лабораторної роботи є розробка одноекранного Android-застосунку з динамічним списком на основі Jetpack Compose, реалізація додавання елементів через форму введення, керування станом кожного елемента та відображення реактивної загальної суми у заголовку з використанням архітектурного компонента ViewModel і StateFlow.

Завдання

Розробити Android-додаток «Витрати за день» для запису і відстеження щоденних фінансових витрат.

Функціональні вимоги:

1. Два поля введення: назва витрати і сума.
2. Кнопка «Додати» активна лише якщо обидва поля заповнені і сума > 0.
3. LazyColumn відображає список витрат з назвою і сумою.
4. Довгий тап на картку відкриває AlertDialog для підтвердження видалення.
5. Загальна сума витрат у заголовку оновлюється реактивно.
Хід роботи

Під час виконання лабораторної роботи було створено Android-проєкт у середовищі Android Studio з використанням Kotlin та Jetpack Compose.

Для збереження інформації про витрати було створено модель Expense, яка містить ідентифікатор витрати, назву та суму.

Для управління станом застосунку було створено клас ExpenseViewModel. У ньому використано MutableStateFlow для зберігання списку витрат. Також реалізовано похідний потік totalAmount, який обчислює загальну суму витрат за допомогою sumOf.

Головний екран застосунку реалізовано у файлі ExpenseScreen.kt. На екрані розміщено два поля введення, кнопку додавання, заголовок із загальною сумою та список витрат. Для списку використано компонент LazyColumn.

Окрема картка витрати реалізована у файлі ExpenseCard.kt. Вона відображає назву витрати та суму. Для довгого натискання використано combinedClickable. Після довгого натискання відкривається AlertDialog, у якому користувач може підтвердити або скасувати видалення витрати.

Результат роботи

У результаті виконання лабораторної роботи було створено одноекранний Android-застосунок «Витрати за день».

Додаток дозволяє:

- вводити назву витрати;
- вводити суму витрати;
- додавати витрату до списку;
- переглядати список витрат;
- автоматично бачити загальну суму витрат;
- видаляти витрати після підтвердження через діалогове вікно.
  

Висновок

Під час виконання лабораторної роботи було розроблено Android-застосунок з використанням Jetpack Compose. Було закріплено навички створення інтерфейсу користувача, роботи з динамічними списками через LazyColumn, обробки введення користувача та реалізації діалогових вікон.

Також було використано архітектурний компонент ViewModel, який дозволяє відокремити логіку застосунку від інтерфейсу. Для реактивного оновлення даних застосовано StateFlow, завдяки чому список витрат і загальна сума автоматично оновлюються після додавання або видалення елементів.
